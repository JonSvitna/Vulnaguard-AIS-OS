# Phase 7: Validation

**Objective:** Confirm that remediated items have been resolved. Scope is defined in the SOW.

---

## Entry Criteria

- Client has submitted evidence of remediation for one or more findings
- Validation testing authorized in rules of engagement (for technical re-testing)

---

## Scope Note

Validation scope is typically limited to the findings identified in the original engagement. A full reassessment of all systems is a separate engagement.

---

## Activities

### 7.1 Technical Re-Testing (VA Engagements)
- Re-test each remediated finding against its original description
- Use the same tooling and methodology used in the original assessment
- Document the re-test result: Resolved / Not Resolved / Partially Resolved

### 7.2 Documentation / Control Validation (Compliance Engagements)
- Review updated policies, procedures, or configurations submitted as evidence
- Confirm the evidence directly addresses the control gap identified
- Interview relevant stakeholders if evidence is ambiguous

### 7.3 POA&M Status Update
- Update each reviewed item with: validation date, result, and reviewer
- Items confirmed resolved → mark Closed
- Items not resolved → keep Open, add a note with specific gaps remaining

### 7.4 Draft Validation Memo
- 1–2 page summary of what was re-tested, what passed, what remains open
- Include updated risk posture compared to original assessment
- Not a full report — a concise memo

---

## Outputs

- Re-test documentation
- Updated POA&M
- Validation memo

---

## Deliverables to Client

- Validation memo
- Final POA&M with updated statuses

---

## Quality Gate

- [ ] All submitted remediation evidence reviewed
- [ ] Every re-tested item has a documented result
- [ ] All Critical findings verified resolved or formally accepted as risk
- [ ] Client has signed the updated POA&M or acknowledged the validation memo

---

## Common Issues

**Finding re-appears after being marked resolved:** Re-open the POA&M item. Note in the memo that the finding was temporarily closed but recurred. This is not unusual.

**Client claims a finding is fixed but submits no evidence:** Request evidence before updating status. "The IT team said it's done" is not sufficient.

**Validation scope creep:** Stick to re-testing items on the POA&M. If new issues are discovered during re-testing, note them as observations in the memo — do not add them to the original report without a separate engagement or change order.
