# Phase 2: Planning

**Objective:** Translate discovery outputs into a structured project plan, confirm access requirements, and run the kickoff meeting.

---

## Entry Criteria

- Phase 1 quality gate passed (written scope confirmation received)
- Discovery outputs documented and reviewed

---

## Activities

### 2.1 Build Project Schedule
- Map each phase to calendar dates based on SOW timeline
- Identify dependencies (e.g., client must provide access before scanning begins)
- Flag any schedule risks (holidays, client team availability, access provisioning lead time)
- Share draft schedule with client for confirmation

### 2.2 Finalize Rules of Engagement (Technical Assessments Only)
- Draft ROE based on confirmed scope
- Include authorized IPs, testing windows, emergency contacts
- Send to client for signature
- **Do not begin any scanning until signed ROE is received**

### 2.3 Prepare Kickoff Agenda
- Introductions
- Engagement overview and objectives
- Scope confirmation
- Timeline and milestones
- Communication cadence and status reporting
- Access and data collection requirements
- Roles and responsibilities
- Q&A

### 2.4 Run Kickoff Meeting
- Send agenda 48 hours in advance
- Record the meeting (with client permission)
- Document all action items and owners
- Confirm escalation path for issues discovered during the engagement

### 2.5 Confirm Access Requirements
- VPN credentials / network access for technical assessments
- SharePoint / document repository access for evidence collection
- Interview scheduling with all identified stakeholders

---

## Outputs

- Signed rules of engagement (technical assessments)
- Project schedule (shared with client)
- Kickoff meeting notes and action items
- Access credentials / provisioning confirmed

---

## Deliverables to Client

- Kickoff agenda (48 hours before the meeting)

---

## Quality Gate

- [ ] Signed ROE on file before any technical testing
- [ ] Client has confirmed project schedule
- [ ] All access requirements identified with owner and ETA
- [ ] Kickoff meeting completed and notes distributed within 24 hours

## Common Issues

**ROE not signed in time:** Do not begin scanning. Escalate to client's primary contact and their leadership if needed. Document the delay.

**Access provisioning delayed:** Update the project schedule and notify client in writing. Adjust downstream milestones accordingly.

**Kickoff attendees can't all make the same time:** Send agenda and recording to absent stakeholders. Do not restart the kickoff — document who attended and who was absent.
