# Security Program Assessment Runbook

**Service Code:** SPA
**Typical Duration:** 3–5 weeks
**Phases:** Discovery → Planning → Assessment → Analysis → Reporting → Closeout

---

## Service Description

A Security Program Assessment evaluates the maturity and effectiveness of an organization's overall security program — governance, policies, risk management, technical controls, and culture. This is a holistic review benchmarked against an established framework (CIS Controls v8 or NIST CSF 2.0).

The output is a maturity scorecard, findings narrative, and a prioritized roadmap to improve program effectiveness.

---

## When to Use This Service

- Organizations without a specific compliance driver who want to understand their overall security posture
- Organizations preparing for cyber insurance renewal
- Organizations building or rebuilding a security program from scratch
- Post-incident reviews where the root cause is program-level (not just technical)

---

## Assessment Framework Options

Select with client at kickoff:

- **CIS Controls v8** — Implementation Group 1 (IG1) is the recommended baseline for most SMBs. IG2/IG3 for larger organizations.
- **NIST CSF 2.0** — Good for organizations with board-level engagement; maps to business risk language.

---

## Discovery Additions

- [ ] Org chart and reporting structure for IT/security roles
- [ ] Current security policies (all)
- [ ] Risk register (if exists)
- [ ] Incident history (past 24 months)
- [ ] Cyber insurance policy and questionnaire (if applicable)
- [ ] Vendor/third-party inventory
- [ ] Training records

---

## Assessment Checklist

### Governance Review
- [ ] Security policy documentation completeness
- [ ] Policy review cadence and ownership
- [ ] Security roles and responsibilities defined
- [ ] Board / executive security reporting cadence
- [ ] Security budget allocation

### Risk Management
- [ ] Risk assessment process in place
- [ ] Risk register maintained
- [ ] Risk acceptance process defined
- [ ] Vendor risk management

### Technical Controls Review
- [ ] Inventory of hardware and software assets
- [ ] Vulnerability management program
- [ ] Patch management process
- [ ] Access control practices
- [ ] Endpoint protection
- [ ] Network monitoring and detection
- [ ] Backup and recovery capability

### Incident Response
- [ ] Incident response plan documented
- [ ] IR roles and contact tree defined
- [ ] IR plan tested (tabletop or live)
- [ ] Incident notification obligations understood

### Security Awareness
- [ ] Security awareness training program
- [ ] Phishing simulation cadence
- [ ] New employee security onboarding

### Stakeholder Interviews
- [ ] IT Director / CISO
- [ ] HR (for awareness and onboarding)
- [ ] Legal / Compliance (for policy and regulatory context)
- [ ] Business Owner / Executive (for risk appetite and security culture)

## Maturity Scoring

Score each control domain on a 1–5 maturity scale:

| Level | Label | Description |
|---|---|---|
| 1 | Initial | Ad hoc, undocumented, reactive |
| 2 | Developing | Some documentation, inconsistently applied |
| 3 | Defined | Documented, consistently applied |
| 4 | Managed | Measured, monitored, improving |
| 5 | Optimizing | Continuously improved, industry-leading |

Present results as a maturity heatmap (by domain) and an overall maturity score.
